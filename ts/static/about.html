How to use it:

The session is an equivalent to the set of data.

New session ID is created as a MD5-hash of user address, time of data entering and s random number when the new data are entered.

The session ID is stored in cookie so the data entered are remembered for the user until been changed.

It is possible to recall an old session if you know the session ID for it.
